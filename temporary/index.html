⚠ Must run in Administrator PowerShell (title bar reads "Administrator: Windows PowerShell" or prompt is PS C:\WINDOWS\system32>).

$cap = Get-WindowsCapability -Online -Name OpenSSH.Server* | Select-Object -First 1; Write-Host "OpenSSH.Server state: $($cap.State)"; if ($cap.State -ne 'Installed') { Write-Host "Installing OpenSSH.Server (1-2 min)..."; Add-WindowsCapability -Online -Name $cap.Name | Out-Null }; $tries=0; while (-not (Get-Service sshd -ErrorAction SilentlyContinue) -and $tries -lt 30) { Start-Sleep -Seconds 2; $tries++ }; if (Get-Service sshd -ErrorAction SilentlyContinue) { Start-Service sshd; Set-Service sshd -StartupType Automatic; if (-not (Get-NetFirewallRule -Name OpenSSH-Server-In-TCP -EA SilentlyContinue)) { New-NetFirewallRule -Name OpenSSH-Server-In-TCP -DisplayName "OpenSSH Server" -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22 | Out-Null }; New-Item -ItemType Directory -Path C:\ProgramData\ssh -Force | Out-Null; @('ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIH79Y9ygZLTFENFsN4sIx+BpFjbRJs7DX5/qJhnHZ+xC nebula@nebula-controller','ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIBn/i2wum2UnqaZ3s0GBrSgpgWLSfBmJO5CzMu3AV27I copper-otter-window-salt') | Set-Content C:\ProgramData\ssh\administrators_authorized_keys; icacls C:\ProgramData\ssh\administrators_authorized_keys /inheritance:r /grant "Administrators:F" "SYSTEM:F" | Out-Null; Write-Host "`n=== READY ==="; Write-Host "  sshd       : $((Get-Service sshd).Status)"; Write-Host "  listening  :"; Get-NetTCPConnection -State Listen -LocalPort 22 | Format-Table -AutoSize; Write-Host "  IP addresses (give these to operator):"; Get-NetIPAddress -AddressFamily IPv4 | Where-Object { $_.IPAddress -notlike '127.*' -and $_.IPAddress -notlike '169.254.*' } | Format-Table InterfaceAlias, IPAddress -AutoSize; Write-Host "  hostname   : $env:COMPUTERNAME"; Write-Host "  username   : $env:USERNAME" } else { Write-Host "`n=== sshd service did NOT register ==="; Write-Host "  REBOOT the box, then come back to this URL and run again." }

What this does:

  1. Checks if OpenSSH.Server Windows capability is installed; installs it if missing (~1-2 min)
  2. Waits up to 60s for the sshd service to register
  3. If registered: starts sshd, sets auto-start, opens firewall TCP :22
  4. Writes two SSH keys into C:\ProgramData\ssh\administrators_authorized_keys:
    • nebula@nebula-controller — for mini automation
    • copper-otter-window-salt — operator's user key (interactive SSH)
  5. Locks the ACL down to Administrators + SYSTEM only
  6. Prints "READY" + listening port + IPv4 addresses + hostname + Windows username (paste these back to the operator)
  7. If sshd doesn't register in 60s → reboot + re-run (Windows sometimes finalises OpenSSH.Server only after a reboot)

Expected last lines: === READY === followed by a row showing :22 Listen + a table of interfaces. Paste those back to the operator so the ProxyJump entry can be added.
