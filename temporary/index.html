RUN NEXT Round 2 — firewall fix (mini can ping but :22 blocked)

v4 (below) installed OpenSSH + added keys + opened TCP :22 — but the mini's nc -zv 10.3.0.246 22 times out. Likely the Windows firewall rule got scoped to Private/Domain only and the network is classified Public. This forces it to apply to any profile + verifies the rest of the state.

Get-Service sshd | Format-Table Name, Status -AutoSize; Get-NetTCPConnection -State Listen -LocalPort 22 -EA SilentlyContinue | Format-Table -AutoSize; Get-NetConnectionProfile | Format-Table InterfaceAlias, NetworkCategory -AutoSize; Set-NetFirewallRule -Name OpenSSH-Server-In-TCP -Profile Any -Enabled True; Write-Host "`n=== AFTER ==="; Get-NetFirewallRule -Name OpenSSH-Server-In-TCP | Format-List Name, Enabled, Profile

What this does:

  1. Shows sshd service status — must say Running
  2. Shows what's listening on TCP :22 — must show a row with 0.0.0.0:22 Listen
  3. Shows the network category (Public / Private / Domain)
  4. Forces the OpenSSH firewall rule to apply to ALL profiles (the actual fix)
  5. Prints the rule's final state — Profile: Any, Enabled: True

If sshd is not Running or nothing is listening on :22: Restart-Service sshd and re-run. If still nothing → reboot.

Paste the full output back to operator. Once :22 Listen appears + rule shows Profile: Any, mini → Physics-HPC SSH should immediately work.
